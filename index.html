Q2. What is a Main method in c# program ?

Main Method

  • The Main() method is the entry point a C# program from where the execution starts.
  • Main() method must be static because it is a class level method. To be invoked without any instance of the class it must be static. Non-static Main() method will give a compile-time error.
  • Main() Method cannot be overridden because it is the static method. Also, the static method cannot be virtual or abstract.
  • Overloading of Main() method is allowed. But in that case, only one Main() method is considered as one entry point to start the execution of the program.

Q6. What are value types and reference types in c#?

Value Types vs Reference Types

  • Value Types
    • Variables that directly contain values. Example :- int i = 100;
    • Value types : bool,char,decimal,double,enum,float,int,long,sbyte,short,struct,uint,ulong,ushort
  • Reference Types
    • Unlike value types, a reference type doesn't store its value directly. Instead, it stores the address where the value is being stored. In other words, a reference type contains a pointer to another memory location that holds the data. Example :- string s = "Hello World!!";
    • Reference Types : String,Arrays (even if their elements are value types),Class,Delegate
Q7. What is a Null Coalescing Operator? Explain with example.

Null Coalescing Operator

  • By default Value types are non nullable.To make it nullable use '?'. Eg:- int? j=null
  • In C#, ?? operator is known as Null-coalescing operator. It will return the value of its left-hand operand if it is not null. If it is null, then it will evaluate the right-hand operand and returns its result. Or if the left-hand operand evaluates to non-null, then it does not evaluate its right-hand operand.
  • int ? item_1 = null;
    int ? item_2 = item_1 ?? 100;
    int? item_2 = item_1.Value;
    Console.WriteLine("Value of item_2 is: {0}", item_2); //Value of item_2 is: 100
    Here, the commented line will give you an InvalidOperationException. So to overcome this problem we use ?? operator

Q27. What is Inheritance? Explain with an Example?

Inheritance

  • If we want to use some features from a Parent class to a Child class , we use Inheritance.

  • We can specify all the common fields, properties, methods in the base class which allows reusability .
  • In derived class we will have only fields, properties, methods, that are specific to them.
  • Program
    namespace Programs{
    class ParentClass{
    public void ParentClassMethod(){
    Console.WriteLine("Parent Class Method Called.");
    }
    }
    class ChildClass : ParentClass{
    public void ChildClassMethod(){
    Console.WriteLine("Child Class Method Called.");
    }
    }
    class Program{
    static void Main(){
    ChildClass C1 = new ChildClass();
    C1.ParentClassMethod();
    C1.ChildClassMethod();
    Console.ReadKey();
    }
    }
    }

Q29. What is Abstraction? Explain with an Example?

Abstraction

  • Abstraction is the process of identifying only the required characteristics of an object ignoring the irrelevant details.
  • Program
    abstract class Shape{
    public abstract int Area();
    }
    class Square : Shape{
    private int Side;
    public Square(int side){
    Side = side;
    }
    public override int Area(){
    return Side * Side;
    }
    }
    class Program{
    static void Main(string[] args){
    Shape shape = new Square(4);
    Console.WriteLine("Area of square is {0}.", shape.Area());
    }
    }
  • Abstraction vs Encapsulation
    • Encapsulation is data hiding(information hiding) while Abstraction is detail hiding(implementation hiding).
    • While encapsulation groups together data and methods that act upon the data, data abstraction deal with exposing to the user and hiding the details of implementation.

Q31. What is the difference between Method Overriding and Method Hiding?

Method Overriding vs Method Hiding

  • In Method Overriding a BaseClass reference variable pointing to a child class object will invoke the overridden method in the ChildClass.

Q33. What are Properties in C#?

Properties

  • Properties are the special type of class members that provides a flexible mechanism to read, write, or compute the value of a private field.
  • Properties can be used as if they are public data members, but they are actually special methods called accessors. This enables data to be accessed easily and helps to promote the flexibility and safety of methods.
  • Encapsulation and hiding of information can also be achieved using properties. It uses pre-defined methods which are “get” and “set” methods which help to access and modify the properties.
